33  bits : int

35  eps : float
36  The smallest representable positive number such that
37  ``1.0 + eps != 1.0``. Type of `eps` is an appropriate floating

39  epsneg : floating point number of the appropriate type
40  The smallest representable positive number such that
41  ``1.0 - epsneg != 1.0``.
42  iexp : int
43  The number of bits in the exponent portion of the floating point
44  representation.
45  machar : MachAr
46  The object which calculated these parameters and holds more
47  detailed information.
48  machep : int
49  The exponent that yields `eps`.
50  max : floating point number of the appropriate type
51  The largest representable number.
52  maxexp : int
53  The smallest positive power of the base (2) that causes overflow.
54  min : floating point number of the appropriate type
55  The smallest representable number, typically ``-max``.
56  minexp : int
57  The most negative power of the base (2) consistent with there
58  being no leading 0's in the mantissa.
59  negep : int
60  The exponent that yields `epsneg`.
61  nexp : int
62  The number of bits in the exponent including its sign and bias.
63  nmant : int
64  The number of bits in the mantissa.
65  precision : int
66  The approximate number of decimal digits to which this kind of
67  float is precise.
68  resolution : floating point number of the appropriate type
69  The approximate decimal resolution of this type, i.e.,
70  ``10**-precision``.
71  tiny : float
72  The smallest positive usable number. Type of `tiny` is an
73  appropriate floating point type.
74 
75  Parameters
76  ----------
77  dtype : float, dtype, or instance
78  Kind of floating point data-type about which to get information.
79 
80  See Also
81  --------
82  MachAr : The implementation of the tests that produce this information.
83  iinfo : The equivalent for integer data types.
84 
85  Notes
86  -----
87  For developers of NumPy: do not instantiate this at the module level.
88  The initial calculation of these parameters is expensive and negatively
89  impacts import times. These objects are cached, so calling ``finfo()``
90  repeatedly inside your functions is not a problem.
